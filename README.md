# こどもトリアージ - 公開手順ガイド

このフォルダの中身を、無料でインターネット上に公開するための手順書です。

専門用語をできるだけ避けて、画面の流れに沿って書いています。途中でつまずいても大丈夫なように、各ステップで「ここで何をしたか」も書いています。

---

## 用意するもの(全部無料)

公開には3つのアカウントを作ります。すべて無料で、所要時間は合計15〜20分程度です。

1. **GitHub アカウント** — コード(プログラムの設計図)を保管する場所
2. **Vercel アカウント** — Webサイトを公開するサービス(サイトを動かしてくれるサーバー)
3. **Node.js** — パソコン上でこのプロジェクトを動かすためのソフト(ローカルで動作確認したい場合のみ必要)

サイトを公開するだけなら、実は **1と2だけでOK** です。Node.jsは「公開前にパソコンで動作を見てみたい」場合だけ必要になります。

---

## ステップ1：GitHubでコードを保管する場所を作る

### 1-1. GitHubアカウントを作る

1. https://github.com を開く
2. 右上の「Sign up」をクリック
3. メールアドレス・パスワード・ユーザー名を入力(ユーザー名は後で公開URLには出ないので何でもOK)
4. メールに届いた認証コードを入力して登録完了

### 1-2. このプロジェクトのファイル一式をGitHubにアップロードする

「Git」というツールを使う方法もありますが、**初めての方はWebブラウザだけで完結する方法**が一番ラクです。

1. GitHubにログインした状態で、画面右上の「+」アイコン → 「New repository」をクリック
2. 「Repository name」に好きな名前を入力(例：`kodomo-triage`)
3. 「Public」が選択されていることを確認(無料Vercelで公開するには公開設定でOK)
4. 一番下の「Create repository」をクリック
5. 次の画面で「uploading an existing file」というリンクをクリック
6. このフォルダ(triage-app)の中にあるファイルを **すべて** ドラッグ&ドロップ
   - **重要**：`triage-app` フォルダごとではなく、**中身のファイルとフォルダ** をアップロードしてください
   - 含めるもの：`app/`, `components/`, `public/`, `package.json`, `tsconfig.json` など
   - 除外してOK：`node_modules/`(もしあれば。サイズが大きいだけで不要)
7. 下の「Commit changes」(緑のボタン)をクリック

これでコードがGitHub上に保管されました。

---

## ステップ2：Vercelで公開する

### 2-1. Vercelアカウントを作る

1. https://vercel.com を開く
2. 「Sign Up」をクリック
3. **「Continue with GitHub」を選ぶ**(さっき作ったGitHubアカウントで連携)
4. 連携の許可画面で「Authorize Vercel」をクリック
5. 個人利用なら「Hobby」プラン(無料)を選択

### 2-2. プロジェクトをインポート

1. Vercelのダッシュボードで「Add New...」→「Project」をクリック
2. さっき作ったGitHubのリポジトリ(`kodomo-triage`)が一覧に出てくるので「Import」をクリック
3. 設定画面が出てきますが、**何も触らずに「Deploy」をクリック** でOK
   - Vercelが自動で「これはNext.jsのプロジェクトだな」と認識して、必要な設定をしてくれます

### 2-3. 完了！

1〜3分待つと、「Congratulations!」の画面とともに公開URLが発行されます。

例：`https://kodomo-triage.vercel.app`

このURLが、世界中の誰でもアクセスできる本番URLです。

## ステップ3：(任意)独自ドメインを設定する

`vercel.app` 付きのURLでも十分使えますが、`kodomo-triage.com` のような独自ドメインにしたい場合は、ドメイン取得サービス(お名前.com、Google Domains、Cloudflareなど)で年額1,000〜2,000円程度で取得できます。

取得後、Vercelの「Settings」→「Domains」で独自ドメインを追加し、画面の指示に従って設定します。

ただし、本番公開を急ぐ必要がなければ、まずは `vercel.app` のURLで公開して使い勝手を確かめるのがおすすめです。

---

## ステップ4：(できれば)コードを更新する方法

公開後にプロフィール画像を変えたいとか、文言を直したいとか、そういうときの方法です。

### 一番ラクなやり方：GitHubのWeb画面で直接編集

1. GitHubのリポジトリ画面で、編集したいファイルをクリック
2. 右上の鉛筆アイコン(Edit)をクリック
3. ブラウザ上で直接編集
4. 下の「Commit changes」をクリック
5. **数十秒後にVercelが自動で再デプロイ** され、本番URLに反映されます

これだけです。GitHubに変更を保存するたびに、Vercelが自動でビルドして公開してくれる仕組みになっています。

## (発展)パソコンでローカル動作確認したい場合

公開前に手元で動かして確認したい人向けの手順です。Node.jsをインストールする必要があります。

### Node.jsのインストール

1. https://nodejs.org にアクセス
2. 「LTS」と書いてある方の緑のボタンをクリックしてダウンロード
3. ダウンロードしたインストーラーを実行(Macの場合は `.pkg`、Windowsは `.msi`)
4. 画面の指示に従ってインストール

### プロジェクトを動かす

ターミナル(Macの場合)またはコマンドプロンプト(Windowsの場合)を開き、このフォルダに移動して以下を実行：

```bash
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開くと、サイトが表示されます。

## 公開後にやっておくべきこと

### 1. 連絡先の設定

`app/privacy/page.tsx` の「お問い合わせ」セクションに、運営者の連絡先(メールアドレス等)を追記してください。本番運用には連絡先の明記が必要です。

### 2. アクセス解析(任意)

サイトのアクセス数を見たい場合は Google Analytics を導入するのがおすすめです。Vercelの「Analytics」機能を有効にするだけでも基本的な情報は見られます(無料枠あり)。

### 3. Amazonアソシエイト・プログラム

商品リンクを実際にアフィリエイトとして機能させたい場合は、Amazonアソシエイトの審査を通す必要があります。
- 申込URL：https://affiliate.amazon.co.jp/
- 過去180日以内に3件以上の適格販売が必要(180日以内の審査基準は変更される可能性があります)

### 4. Google Search Consoleへの登録(SEO対策)

Googleで検索されるようにしたい場合は、https://search.google.com/search-console から登録します。

## 困ったときは

- **デプロイが失敗する** → Vercelのログを見ると、何のエラーで失敗したか書いてあります。エラーメッセージをそのまま検索すると大体の解決策が出てきます
- **画面が崩れる** → ブラウザのキャッシュを削除して再読み込み(Mac: ⌘+Shift+R / Win: Ctrl+Shift+F5)
- **GitHubの操作が分からない** → 「GitHub 初心者」で検索すると親切な記事がたくさんあります

## 法令遵守について(重要)

本プロジェクトは以下のガイドラインに配慮して設計されていますが、本番公開前に弁護士や薬事コンサルタントによる最終確認を強くおすすめします。

- **薬機法**：商品紹介で効能効果を一切表示しない構造
- **健康増進法**：症状の改善・予防を暗示する表現を排除
- **医療広告ガイドライン**：特定の医療機関の広告とならないよう、外部の公的検索サービスへの導線のみを提供
- **プログラム医療機器(SaMD)該当性ガイドライン**：個別の病名・罹患可能性は表示せず、一般論としての受診タイミング案内に限定
- **ステマ規制(2023年10月〜)**：アフィリエイト枠に「広告」表記を明示

これらは設計上の配慮であり、法的助言ではありません。
